Core Modules¶

Fonctions utilitaires pour construire des commandes ping selon l’OS utilisé.

src.core.ping.build_ping_command(ip: str, count: int = 3, timeout: int = 2) → list[source]¶
Construit une commande ping adaptée au système d’exploitation

(Windows/Linux).

Paramètres:

  • ip (str) – Adresse IP à pinger.

  • count (int, optional) – Nombre de requêtes ICMP à envoyer. Par défaut 3.

  • timeout (int, optional) – Durée maximale (en secondes) d’attente d’une réponse. Par défaut 2.

Renvoie:

Liste représentant la commande ping à exécuter avec subprocess.

Type renvoyé:

list

src.core.runner.run_scan(machines, use_async=False, threads=10, ports=False)[source]¶

Exécute le scan réseau et, si demandé, le scan des ports ouverts avec Nmap.

Paramètres:

  • machines (list of tuple) – Liste de tuples (nom_machine, ip).

  • use_async (bool, optional) – Si True, utilise asyncio. Sinon, multithread.

  • threads (int, optional) – Nombre de threads pour le scan synchrone.

  • ports (bool, optional) – Si True, lance un scan complet des ports TCP.

Renvoie:

Résultats enrichis au format :

(nom_machine, ip, status, latence, liste_ports)

Type renvoyé:

list of tuple

Module de scan réseau asynchrone basé sur asyncio et ping.

async src.core.scanner_async.async_ping_ip(machine_name, ip)[source]¶

Envoie un ping asynchrone à une adresse IP et retourne le résultat.

Paramètres:

  • machine_name (str) – Nom de la machine (ou IP si inconnu).

  • ip (str) – Adresse IP à tester.

Renvoie:
