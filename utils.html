Utils Modules¶

Utilitaires pour lire et écrire des fichiers CSV liés au scan réseau.

src.utils.csv_utils.read_from_csv(filename)[source]¶

Lit un fichier CSV et extrait les noms de machines et leurs IPs.

Paramètres:

filename (str) – Chemin vers le fichier CSV à lire.

Renvoie:

Liste de tuples (nom_machine, ip).

Type renvoyé:

list of tuple

src.utils.csv_utils.save_to_csv(results, filename='results.csv')[source]¶

Enregistre les résultats de scan dans un fichier CSV.

Paramètres:

  • results (list of tuple) – Résultats à enregistrer, sous forme (nom, ip, statut, latence, ports), où ports est une liste de tuples (port, service).

  • filename (str, optional) – Nom du fichier de sortie. Par défaut « results.csv ».

Renvoie:

None

Configuration d’un logger personnalisé pour l’application de scan réseau.

src.utils.logger.setup_logger(name='network_scanner', level=20)[source]¶

Configure et retourne un logger personnalisé avec un format lisible.

Paramètres:

  • name (str, optional) – Nom du logger. Par défaut « network_scanner ».

  • level (int, optional) – Niveau de log (ex: logging.INFO, logging.DEBUG). Par défaut INFO.

Renvoie:

Instance de logger configurée.

logging.Logger

Utilitaires de parsing pour l’extraction de latence et la résolution DNS.

src.utils.parsing.extract_latency(stdout_str)[source]¶

Extrait la latence moyenne d’un retour de commande ping.

Fonctionne pour Linux/macOS, Windows FR/EN, avec prise en charge de

plusieurs formats.

Paramètres:

stdout_str (str) – Sortie brute du ping.

Renvoie:

Latence moyenne en millisecondes, ou None si introuvable.

Type renvoyé:

int or None

src.utils.parsing.resolve_hostname_if_needed(machine_name, ip)[source]¶

Résout le nom DNS de l’hôte si machine_name est une adresse IP brute.

Paramètres:

  • machine_name (str) – Nom ou IP de la machine.

  • ip (str) – Adresse IP de la machine.

Renvoie:

Nom de la machine si résolu, sinon retourne l’IP.

Type renvoyé:

str
